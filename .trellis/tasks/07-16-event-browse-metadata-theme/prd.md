# 完善品牌、地区、标签与夜间模式

## Goal

让公开站点使用“野活网”品牌，优化地区选择交互，建立“投稿标签建议”和“正式规范标签”之间清晰的审核边界，并让全站自动适配系统浅色/深色模式。

## Background

- 公共布局当前在页眉显示字母 Logo `E` 和品牌名 `Eventlist`，页脚及页面标题也仍使用旧品牌。
- 地区选择由 `src/lib/divisions.ts` 和 `src/components/DivisionPicker.svelte` 统一提供。香港、澳门、台湾维持当前占位数据，本次不扩充详细行政区划。
- 省、市、区/县使用三级联动；当前选择省份会清空市和区/县，直辖市不会自动选择其唯一的市级节点。
- 浏览地区的三个可访问标签当前由“浏览地区”拼接为“浏览地区省份 / 城市 / 区县”。
- 投稿表单当前使用 `TagInput.svelte`，允许输入或选择标签；后端 `insertSubmission()` 会立即创建缺失标签并关联到待审核活动。
- 正式活动列表、详情和筛选已经使用规范化的 `tags` / `event_tags` 关系，后台也已有标签归并功能，但发布动作当前不保证活动至少拥有一个规范标签。
- `src/styles/tokens.css` 已定义 `prefers-color-scheme: dark` 的主题令牌；需要验证并补齐所有公共页、后台页和交互组件在系统深色模式下的实际表现。
- 活动当前只保存必填的开始日期和结束日期；投稿、后台编辑、活动卡片、详情页和 JSON-LD 都没有具体时间字段。

## Requirements

### R1 品牌统一

- 移除公共页眉中的字母 Logo `E` 及其装饰块。
- 将产品界面中的 `Eventlist` 品牌文案统一改为“野活网”，包括公共布局中的站点标题、页眉和页脚，以及后台布局、登录页等可见品牌位置；开发样例活动名称不属于品牌界面文案。
- 保留现有导航结构和路由。

### R2 直辖市联动

- 选择北京、上海、天津、重庆等直辖市省级项时，自动选择其对应市级项；用户仍需在投稿/编辑场景选择具体区/县。
- 浏览筛选场景应允许停留在省或市级范围，并保持现有 `division_code` URL 查询契约。

### R3 地区文案精简

- 将浏览地区三个选择器的可访问/可见文字从“浏览地区省份 / 浏览地区城市 / 浏览地区区县”改为“省 / 市 / 区/县”。
- 相关占位文字使用同一套简短术语，避免“区县”和“区/县”混用。

### R4 投稿建议与规范标签分离

- 投稿人只需用自由文本描述大概标签，不应因为投稿而直接创建公开可筛选的规范标签。
- 管理员审核或编辑活动时，将投稿建议映射为已有规范标签，或创建/归并为新的规范标签。
- 已发布活动的标签必须来自统一的规范标签集合。
- 每个活动发布时必须至少关联一个规范标签；没有规范标签的待审核活动不能通过发布动作。
- 公共活动列表、详情和首页标签入口继续展示规范标签，并可按不同规范标签筛选活动。
- 标签筛选使用明确的规范标签匹配，避免一个标签名意外匹配另一个标签名的子串。

### R5 自动夜间模式

- 全站跟随系统 `prefers-color-scheme` 自动切换浅色和深色主题。
- 不增加手动主题切换按钮或持久化偏好。
- 公共页、投稿页、活动页、后台页和弹层/选择器均使用语义主题令牌，并保持文字、边框、焦点态可读。

### R6 可选活动时间

- 保留开始日期和结束日期为必填项。
- 允许投稿人和管理员分别填写可选的开始时间与结束时间，使用活动所在地的本地时间，不进行时区换算。
- 开始时间和结束时间可以独立填写，不要求同时存在。
- 时间使用 24 小时制 `HH:MM` 保存和展示。
- 当活动开始、结束日期相同且两个时间都存在时，结束时间不得早于开始时间；跨日活动按各自日期解释。
- 活动卡片、详情页、后台列表和结构化数据在有时间时显示日期与时间；没有时间的历史活动保持现有仅日期展示。
- 日期筛选继续按日期工作，不因可选时间改变现有 URL 查询参数和活动是否过期的判断。

### R7 合并并重建数据库基线

- 网站尚未部署，不保留增量 migration 历史；将当前 schema、种子数据、审计表、标签建议和时间字段合并到单一 `0001_init.sql`。
- 删除其余 migration 文件，确保新环境一次迁移即可得到完整数据库。
- 以当前代码为准优化约束和索引，不重新引入未被应用使用的 `cities` 表。
- 标签名称在数据库层限制为空白清理后的 1–24 字符，并以不区分 ASCII 大小写的方式唯一。
- 日期、时间、状态和 `tag_suggestions` 长度在数据库层具有明确 CHECK 约束。

## Constraints

- 保持 Astro + Svelte 5 + Bits UI + Tailwind v4 技术栈。
- 保持现有表单字段、路由和主要 URL 查询参数兼容；标签建议和可选活动时间使用向后兼容的 D1 migration 持久化。
- 不引入 Playwright；使用现有构建、类型、lint 和轻量浏览器验证流程。
- 本次不新增或扩充香港、澳门、台湾的详细行政区划数据。
- 不开始实现，直到 PRD、`design.md` 和 `implement.md` 完成并经用户批准。

## Acceptance Criteria

- [x] AC1：公共页眉不再显示字母 Logo，品牌显示为“野活网”；所有面向用户的旧品牌文案已完成一致性检查。
- [x] AC2：选择四个直辖市之一后，市级项自动填充；切换到非直辖市时不会残留旧市/区县值。
- [x] AC3：浏览地区三个控件显示/朗读为“省”“市”“区/县”。
- [x] AC4：公开投稿自由文本不会直接污染规范标签集合。
- [x] AC5：管理员能在审核流程中将投稿建议整理为至少一个规范标签；零规范标签的活动不能发布或重新发布。
- [x] AC6：已发布活动的标签可在列表、详情、首页入口展示，并可通过精确标签筛选得到正确活动。
- [x] AC7：操作系统切换浅色/深色模式后，全站自动适配，无需刷新或手动开关；两种模式均通过构建、lint、类型和关键页面视觉检查。
- [x] AC8：投稿和后台编辑可独立保存可选开始/结束时间；同日完整时间范围通过顺序校验；公开页面和 JSON-LD 在有时间时正确展示，没有时间时保持日期展示兼容。
- [x] AC9：全新空数据库只应用 `0001_init.sql` 即可通过 schema、种子数量、外键、CHECK 和应用构建验证；仓库中不再保留后续 migration 文件。
