# 活动页查询已结束活动

## Goal

允许用户在活动浏览页主动查询已经结束的活动，而不影响当前默认展示未结束活动的行为。

## Background

- 当前活动页通过 URL 查询参数承载地区、类型、日期、规模、标签、排序和分页筛选。
- 当前公开活动查询无条件要求 `end_date` 不早于当天，因此活动页无法列出历史活动。
- 已结束活动详情仍属于公开活动数据，本任务只扩展活动浏览能力。

## Requirements

- 活动浏览页必须提供“未结束 / 已结束 / 全部”三项状态筛选。
- 默认活动列表继续保持当前面向未结束活动的展示逻辑。
- “已结束”按北京时间判断：结束日期早于当天，或结束日期为当天且已超过明确的结束时间；未填写结束时间时在当天结束前仍视为未结束。
- “全部”不按结束日期或结束时间排除已发布活动。
- 已结束活动仍须遵守现有公开可见性、地区、标签、分页等适用筛选规则。
- 查询“已结束”活动且用户未显式指定排序时，必须优先展示最近结束的活动。
- 用户显式选择排序后，状态切换、其他筛选和分页必须保留该排序。

## Acceptance Criteria

- [x] 用户可从活动浏览页查询并看到已结束活动。
- [x] 用户可在“未结束 / 已结束 / 全部”之间切换，且状态保存在 URL 查询参数中。
- [x] 未主动查询已结束活动时，默认列表行为保持不变。
- [x] “已结束”列表默认按结束日期从近到远排列。
- [x] 北京时间当天已超过 `end_time` 的活动会进入“已结束”，当天未填写 `end_time` 的活动仍留在“未结束”。
- [x] 查询已结束活动时，现有适用筛选条件和分页行为正常工作。
- [x] 切换状态或移除其他筛选时会重置页码，分页链接会保留当前状态。
- [x] 页面标题、说明、空状态和已启用筛选能准确反映当前状态。
- [x] Prettier、TypeScript 类型检查和生产构建通过；ESLint 按项目 spec 中记录的 TypeScript 7 上游兼容例外处理。

## Out of Scope

- 不修改首页仍只展示未结束活动的行为。
- 不修改活动发布状态、审核流程或活动详情页可见性。
- 不新增数据库字段或迁移。
